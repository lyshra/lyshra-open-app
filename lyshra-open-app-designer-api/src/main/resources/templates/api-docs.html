Quick Start

1. Authenticate
POST /api/v1/auth/login
Content-Type: application/json

{
  "username": "your_username",
  "password": "your_password"
}
2. Use the Token
GET /api/v1/workflows
Authorization: Bearer <your_jwt_token>
Content-Type: application/json

API Endpoints

Authentication

User login, registration, and token management

POST /api/v1/auth/login
POST /api/v1/auth/register
POST /api/v1/auth/refresh
Workflows

Workflow definition CRUD operations

GET POST /api/v1/workflows
GET PUT DEL /api/v1/workflows/{id}
PUT /api/v1/workflows/{id}/state
Versions

Version management and activation

GET POST /api/v1/workflows/{id}/versions
PUT /versions/{id}/activate
PUT /workflows/{id}/rollback
Executions

Execute and monitor workflows

GET /api/v1/executions
POST /executions/workflows/{id}
GET /executions/stream SSE
Processors

Processor metadata for designer UI

GET /api/v1/processors
GET /processors/{identifier}
GET /processors/categories
Validation & Schemas

Validate workflows and access schemas

POST /api/v1/validation/workflow
GET /api/v1/schemas
GET /schemas/bundle

HTTP Status Codes

Code Status Description
200 OK Request successful
201 Created Resource created successfully
204 No Content Resource deleted successfully
400 Bad Request Validation error or malformed request
401 Unauthorized Authentication required or invalid token
403 Forbidden Insufficient permissions
404 Not Found Resource not found
409 Conflict Resource conflict (e.g., duplicate name)
422 Unprocessable Entity Schema validation failed
500 Internal Error Unexpected server error

Error Response Format

{
  "timestamp": "2025-01-25T10:30:00Z",
  "status": 400,
  "code": "VALIDATION_ERROR",
  "message": "Request validation failed",
  "details": {
    "fieldErrors": [
      { "field": "name", "message": "Workflow name is required" }
    ]
  }
}
